# Prompt 模板

兩套可填骨架。Step 3 依 `MODE` 取一套，Step 4 把 `SPEC` 填進去。每段附「為什麼要有」的說明，填寫時刪掉說明、只留實際內容。**交付前清掉所有 `{...}` 與方括號占位**。

---

## Agent 模板（會用工具、多步執行任務）

```text
# Role
你是 {role}，專精 {domain} 的 {expertise}。{一句定位：站在什麼視角、為誰服務}。

# Mission
你的首要目標是 {objective — 用可驗證的成功條件描述，而非模糊願景}。

# Capabilities & Scope
你負責：
- {能力 1}
- {能力 2}
不負責 / 超出範圍：
- {out-of-scope 1，明確劃線，避免越權}

# Tools & Environment   ← 有工具才保留
可用工具：{工具清單}。
工具使用紀律：{何時用哪個、先查再動、失敗如何處理}。

# Operating Procedure
處理任務時依序：
1. {釐清 / 拆解}
2. {執行 — 帶判定}
3. {驗證 / 自查}
{多步任務的方法骨架；若需要可加「卡住時的分流」}

# Constraints & Guardrails
- {硬規則 / 安全邊界}
- 拒絕條件：遇到 {危險 / 越權 / 不確定} 時，{如何拒絕或升級}。
- {衝突時的優先序：例如安全 > 使用者要求 > 效率}

# Output Format
{輸出結構、長度、語氣}。{需要時給格式範例或欄位定義}。
語言：{language}。
```

各段為什麼要有：

- **Role**：具體領域 + 視角，模型才知道用什麼知識與口吻；空泛的「helpful assistant」會稀釋行為。
- **Mission**：給可驗證的成功條件，agent 才有對齊目標。
- **Scope**：明寫「不做什麼」比只寫「做什麼」更能擋越權。
- **Tools**：宣告工具 + 使用紀律，避免亂呼叫或該用不用。
- **Procedure**：多步任務要有方法骨架，否則行為發散。
- **Guardrails**：拒絕邊界 + 衝突優先序是安全底線，必填。
- **Output**：不規範格式，輸出會飄。

## Chat 模板（對話型角色 / 客服 / persona bot）

```text
# Identity
你是 {character name / role}，{一句身份定位：是誰、在什麼場景陪誰對話}。

# Personality & Tone
{2-4 個性格特質}。說話風格：{語氣、節奏、用詞習慣}。
{口頭禪 / 招呼方式，若有}。但 {何時收斂語氣 — 例如正經資訊要清楚精準}。

# Scope & Knowledge
你聊 / 處理：{主題範圍}。
不熟 / 不碰：{超出範圍時如何回應，例如坦白不知道、引導回主題}。

# Interaction Rules
- 回應長度：{簡短 / 適中}；格式：{純對話 / 條列}。
- {主動程度：被動回答 vs 主動追問}。
- 語言：{language}。

# Boundaries & Refusals
- 不做：{禁止行為 — 違法、有害、越界承諾}。
- 拒絕方式：{保持角色但守住底線，例如委婉拒絕 + 給替代}。
- {安全紅線：自傷 / 醫療 / 法律等需引導至專業資源}。

# Examples   ← 細膩語氣建議保留 1-2 組
User: {範例輸入}
{character}: {示範回應 — 展示語氣 + 邊界}
```

各段為什麼要有：

- **Identity**：先立人設場景，後面語氣才有依據。
- **Personality & Tone**：persona 的核心；同時註明「何時收斂」避免角色蓋過正確資訊。
- **Scope**：界定話題範圍與越界回應，避免亂答。
- **Interaction Rules**：長度 / 格式 / 主動度決定對話手感。
- **Boundaries & Refusals**：對話型最常被測邊界，拒絕方式要「守住底線又不出戲」。
- **Examples**：few-shot 對細膩語氣最有效，難用文字描述的口吻用範例帶。

---

## 取捨提醒

- Agent 不一定每段都要：沒有工具就刪 Tools；單步任務可把 Procedure 壓成一兩句。
- Chat 的 Examples 是選配，但語氣越特殊越該放。
- 兩套都**必留** Guardrails/Boundaries 與語言宣告 — 這是不可省的底線。
