# /plan [描述]

呼叫 **planner agent** 產出結構化的實作計畫。

## 流程

1. 重述需求（確認理解正確）
2. 識別影響範圍（修改/新增/刪除的檔案）
3. 拆解 Phase + 步驟（帶檔案路徑和原因）
4. 評估風險和依賴關係
5. **等待確認後才開始實作**

## 輸出格式

```text
# 實作計畫：<功能名稱>

## 影響範圍
- 修改：...
- 新增：...

## Phase 1：<名稱>
1. <步驟>（檔案：path/to/file）
   - 動作：...  原因：...  風險：低/中/高

## 風險
| 風險 | 嚴重度 | 緩解 |

等待確認 → 確認後開始實作
```

## 使用時機

- 開始新功能
- 複數檔案受影響的變更
- 需求不明確或有歧義時
