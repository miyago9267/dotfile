# Tests

驗收條件使用 EARS（Easy Approach to Requirements Syntax）語法。

## Phase 1: 基礎建設

### T1.1 安裝腳本

- [x] When 在 macOS 上執行 `install_sops.sh`，then age 和 sops 透過 brew 安裝
- [x] When 在 Arch Linux 上執行 `install_sops.sh`，then age 和 sops 透過 pacman 安裝
- [x] When 在 Ubuntu 上執行 `install_sops.sh`，then age 和 sops 從 GitHub release 下載 binary
- [x] When age 和 sops 已安裝，then 腳本靜默跳過不重複安裝
- [x] When 在不支援的 OS 上執行，then 印出警告並 exit 0（不阻斷安裝流程）

### T1.2 目錄結構

- [x] `secrets/.sops.yaml` 存在且指定正確的 age public key
- [x] `secrets/tokens.enc.yaml` 存在且為有效的 sops 加密格式
- [x] `secrets/README.md` 存在且包含初始化、新增 secret、換機器的說明
- [x] `.gitignore` 包含 `secrets/*.dec.*` 排除規則

## Phase 2: Shell 整合

### T2.1 自動載入

- [x] When `~/.age/key.txt` 不存在，then `secrets.zsh` 靜默跳過（return 0）
- [x] When sops 未安裝，then `secrets.zsh` 靜默跳過（return 0）
- [x] When 加密檔不存在，then `secrets.zsh` 靜默跳過（return 0）
- [x] When 快取檔存在且比加密檔新，then 直接 source 快取檔（不呼叫 sops）
- [x] When 快取檔不存在或比加密檔舊，then 呼叫 sops 解密並產生新快取
- [x] When 解密失敗，then 靜默跳過（不阻斷 shell 啟動）

### T2.2 快取機制

- [x] 快取檔 `~/.env.secrets` 權限為 600
- [x] 快取內容格式為 `export CATEGORY_KEY="value"`
- [x] YAML 巢狀一層正確展開：`github.GITHUB_TOKEN` 轉為 `GITHUB_GITHUB_TOKEN`

### T2.3 Helper functions

- [x] `secrets-edit` 呼叫 sops 開啟加密檔案，設定 `SOPS_AGE_KEY_FILE`
- [x] `secrets-reload` 刪除快取檔後重新 source secrets.zsh
- [x] `secrets-init` 產生 age key pair（如不存在）、更新 .sops.yaml、首次加密模板

## Phase 3: Setup 整合

### T3.1 setup.sh 整合

- [x] `setup.sh` 的安裝清單包含 `install_sops.sh` 項目

### T3.2 sec CLI 工具

- [x] `sec init` 產生 age key pair 並更新 .sops.yaml
- [x] `sec edit` 以 sops 開啟加密檔（vim 時設定 ft=yaml）
- [x] `sec show` 解密並顯示完整 secret（支援 bat 上色）
- [x] `sec list` 只顯示 key 名稱，不洩漏 value
- [x] `sec status` 顯示 age/sops 版本、key 路徑、快取狀態、加密檔狀態
- [x] `sec push` 將加密檔 commit + push；無變更時提示並跳過
- [x] `sec export <file>` 匯出 yaml 或 .env 格式，檔案權限 600
- [x] `sec import <file>` 從明文 yaml 匯入、備份現有加密檔、清快取
- [x] `sec reload` 清快取並重新解密寫入 `~/.env.secrets`
- [x] When 無參數執行 `sec`，then 顯示 usage 說明
- [x] When age key 不存在，then 管理指令報錯並提示 `sec init`
- [x] When sops 未安裝，then 報錯並提示安裝路徑

### T3.3 Onboarding 自動引導

- [ ] When 首次執行 `setup.sh` 且 `~/.age/key.txt` 不存在，then 自動引導使用者選擇建立新 key 或匯入現有 key
- [ ] When 使用者選擇建立新 key，then 執行 `age-keygen` 並更新 `.sops.yaml`
- [ ] When 使用者選擇匯入 key，then 提示輸入 key 路徑或從剪貼簿貼上

### T3.4 PowerShell 整合

- [ ] `setup.ps1` 包含 age + sops 安裝流程（透過 scoop 或 winget）
- [ ] PowerShell profile 包含 secret 載入模組（對應 secrets.zsh 功能）
