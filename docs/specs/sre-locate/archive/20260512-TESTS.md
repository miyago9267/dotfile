## 驗收條件 (EARS)

### R1: 平行定位

- **When**: Miyago 提供一個 keyword
- **Shall**: 平行掃過所有 kubectl contexts + gcloud projects，回報命中位置與 hit count
- **驗證方式**: 用一個已知 service name（例：在 develop cluster 部署的服務）跑 `locate.sh <name>`，輸出應包含該 cluster/ns，且 hit count > 0
- **狀態**: [ ] 未驗證 / [x] 通過

### R4: 部分失敗容錯

- **When**: 某個 context 認證過期或不可達
- **Shall**: 跳過該 source 但繼續其他 source，輸出標記失敗來源
- **驗證方式**: 暫時 unset 某個 cluster credential（或用假 context name），確認其他 cluster 結果仍正常回傳，且失敗 source 在輸出有明確標記
- **狀態**: [ ] 未驗證 / [x] 通過

### R5: 結果截斷

- **When**: 結果筆數超過 limit
- **Shall**: 截斷並提示加 filter
- **驗證方式**: 用超寬 keyword（例：`a`），確認每 source 結果不超過 20 筆，且輸出含「結果已截斷」提示
- **狀態**: [ ] 未驗證 / [x] 通過

### R6: Read-only

- **When**: skill 被觸發
- **Shall**: 不執行任何寫操作
- **驗證方式**: code review locate.sh，grep `apply|delete|patch|edit|scale|create|set`，應無寫操作動詞
- **狀態**: [ ] 未驗證 / [x] 通過

### R2 / R3: 留到後續 Phase
